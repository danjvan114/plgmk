# KNExpanse 扩展开发规范
> 适用版本：KNExpansev1.2.2以上
> 一个 KNExpanse 扩展 =**一个 `.cue.js` 文件**，用 **JavaScript** 编写，导入后
向积木面板添加自定义积木。--
## 1. 用什么写（编辑器）
任意**纯文本编辑器**即可，推荐：- **VS Code**（推荐，带语法高亮、括号匹配）- Notepad++ / Sublime Text / Vim- 系统自带的「记事本」也能用，但无高亮，容易看错
⚠
不要用 Word/WPS 等富文本编辑器保存——它们会加隐藏格式，导致文件不是合
法 JS。
保存时务必选 **UTF-8 编码**、**纯文本（.txt 另存为后改后缀为 .cue.js）**。--
## 2. 用什么语言写
**JavaScript（JS）。不是 Python，也不是 C++。**
原因：KNExpanse 的扩展运行在浏览器内核（WebView2/Edge）里，和网页一样是 JS
环境。
你写的 `.cue.js` 会被框架当作一段 JS 脚本执行，因此：
✅
❌
能用 JS 的所有语法：变量、函数、Promise、`fetch`、`requestAnimationFrame`、
`document`、`window` 等浏览器 API。
不能写 Python（`def`/`print`）或 C++（`#include` / `cout`）——运行环境根本
不认识。
❌
不能调用本地命令行、不能直接用浏览器 API 读本机文件（受 WebView2 沙箱限
制）。但可以通过 **`window.pywebview.api` 的本地接口**读取本机文件 / 剪贴板 / 目
录（见第 9 节），也能走 `window.pywebview.api.request` 绕过跨域（见第 6 节）。--
## 3. 文件后缀（哪些能用 / 不能用）
导入功能只认 **`.cue.js`**（也兼容普通 `.js`）。明确清单：
| 后缀 | 能否导入 | 说明 |
|------|----------|------|
| **`.cue.js`** |
✅
| **`.js`** |
✅
推荐 | 标准扩展文件，导入扩展功能首选 |
可用 | 能被识别导入，但建议统一用 `.cue.js` 以区分 |
| `.py` (Python) |
❌
| `.cpp` / `.c` (C++) |
| `.txt` |
❌
不行 |Python 不在运行环境，导入会失败 |
❌
不行 | 需要编译，浏览器跑不了 |
不行 | 无 JS 结构，`exports.extension` 不会被识别 |
不行 | 不是扩展脚本格式 |
| `.html` |
| `.json` |
❌
❌
不行 | 只能存数据，不能定义积木逻辑 |
**结论**：写扩展就用 `.cue.js` 一个后缀，最省心。--
## 4. 扩展文件最小模板
```js
// 顶部可选：拿到扩展专用 utils（需要事件 / 反馈时用）
const utils = require('utils');
const MY_EXT = {
type: 'MY_EXT',
扩展重复
title: '我的扩展',
// 唯一标识，大写英文/数字/下划线，不要和已有
// 工具箱里显示的名字
icon: 'data:image/svg+xml;...', // 图标（见第 5 节），也可传 PNG base64
color: '#608FEE',
// 积木主题色（十六进制）
methods: [ /* 这里放积木定义（第 5 节） */],
events: [],
// 事件帽（没有就留空数组）
toolbox: [ /* 工具箱布局（第 5 节） */]
};
// 必须：用 CommonJS 风格暴露扩展对象（不要用 exportdefault/ESM）
exports.extension = MY_EXT;
```
框架用 `newFunction('require','exports', code)` 执行你的文件，所以：- 顶层可以用 `constutils=require('utils')`（返回扩展专用 utils 副本）；不用可不写。- **末尾必须写 `exports.extension = 你的对象`**。- 顶层**不要**写 `return` / 顶层 `await` / `import`（会直接语法/执行报错）。--
## 5. 积木（method）怎么写
```js
{
type: 'my_block',
message0: '计算 %1 加 %2',
args0: [
// 积木唯一 ID
// 文字模板，%1%2... 是字段占位符
{ type: 'input_value', name: 'a', check: 'Number' }, // 值槽
{ type: 'field_input', name: 'b', text: '10' }
// 文本输入框（默认 10）
// { type: 'field_dropdown', name: 'mode', options: [['行数','rows'],['列数
','cols']] } // 下拉
],
output: 'Number',
略 = 命令积木
// 有 output = Reporter（返回值的积木）；省
// previousStatement: null, nextStatement: null, // 命令积木若要上下拼接可加
tooltip: '一句话说明这个积木干什么',
function: function (params) { // 运行时执行的回调，params 是各字段值
return Number(params.a) + Number(params.b); // 同步返回结果
}
}
```
字段 `type` 支持：- `field_input` —— 文本输入框（用户直接打字）。- `field_dropdown` —— 下拉框；`options` 是 `[[显示文字, 值],...]` 二维数组，也可是
一个**同步**函数返回该数组（ 不能返回 Promise）。
⚠- `input_value` —— 值槽（接别的积木的输出）。
**运行时回调 `function` 的两条铁律**：
1
. 框架对 `function` 是**同步调用**、不会 `await`Promise。需要异步（如 `fetch` 拉
网络、定时器）时，用「fire-and-forget+toast 反馈」模式：函数里启动异步任务，直
接 `return null`，任务完成后再用 `window.__CUE_TOAST__` 显示结果。
2
. 给用户反馈统一用：
```js
window.__CUE_TOAST__('提示文字', 'ok' | 'error' | undefined);
```
（`undefined` 是普通提示色，'ok' 绿，'error' 红）
**工具箱 `toolbox` 布局**写法：
```js
toolbox: [
{ type: 'label', text: '分组标题' },
{ type: 'method', block: 'my_block' },
{ type: 'event', block: 'my_event' },
// 分组文字
// 引用 methods 里的积木
// 引用 events 里的事件帽
{ type: 'button', id: 'my_btn', text: '打开某网页 ↗', click: function(){ /* ... */ } } //
顶部真按键
]
```
**事件帽 `events`**（可选）：
```js
events: [
{
type: 'my_event',
text: '当 某事情 发生时',
icon: 'data:image/svg+xml;...',
color: '#608FEE',
// 小图标（约 20x20）
params: [{ type: 'message', text: '错误信息', check: 'String' }] // 参数槽
}
]
```
在积木 `function` 里触发：`utils.emitEvent('my_event', { message: '出错啦' })`。
**图标 `icon`**：传一个 data URL 字符串即可。SVG 内颜色用 `#` 不要写成 `%23`
（否则黑块）。
SVG 示例（蓝色圆形 + 白字）：
```js
function b64utf8(str){const b=new TextEncoder().encode(str);let s='';for(let
i=0;i<b.length;i++)s+=String.fromCharCode(b[i]);return btoa(s);}

积木里想请求网络（如调用编程猫后端）时：

const method = (opts && opts.method || 'GET').toUpperCase();
const body = (opts && opts.body) || null;

}, credentials: 'include', body: body })
.then(r => r.json().catch(() => r.text()))
.catch(function (e) {
（带登录 Cookie）
if
(window.pywebview
// 跨域失败 → 回退桌面壳代理
&&
window.pywebview.api
window.pywebview.api.request) {
const cookie = document.cookie || '';
&&
return window.pywebview.api.request(url, method, body, cookie)

## 7. 怎么导入
1
. 打开 KNExpanse 桌面程序。
2
. 顶部菜单点 **「设置 → 导入扩展...」** 导入扩展；点 **「设置 → 扩展集市」** 可
在桌面壳内打开扩展集市 https://code.pgrm.run/（不调系统浏览器）。
3
. 在弹出的窗口里选择你的 `.cue.js` 文件。
4
. 提示「导入成功」即生效；工具箱出现你的扩展分组，拖出积木即用。
5
. 导入失败会显示红色错误原因（常见：后缀不对、不是合法 JS、缺少
`exports.extension`、与其他扩展 `type` 重名）。--

## 9. 读取本机文件 / 剪贴板 / 目录（本地能力）
扩展运行在 WebView2 沙箱里，JS 本身读不到 `C:/`、`D:/`、桌面、剪贴板。但**桌面
壳（Python 侧）暴露了一组本地读取接口**，扩展通过 `window.pywebview.api` 调用，
由桌面壳在本地执行、结果回传给你。
可用接口（都返回 Promise，用 `.then` 取结果）：
| 接口 | 参数 | 返回 | 说明 |
|------|------|------|------|
| `read_text_file(path, encoding?)` | 路径字符串 | `{ok, content}` / `{ok:false, error}` |
读文本文件（桌面 `C:/Users/你/Desktop/xxx.txt`、D 盘 `D:/data/a.txt` 等） |
| `write_text_file(path, content, encoding?)` | 路径、内容 | `{ok}` / `{ok:false, error}` |
写文本文件（自动建目录） |
| `read_clipboard()` | 无 | `{ok, text}` / `{ok:false, error}` | 读系统剪贴板文本 |
| `list_directory(path?)` | 目录路径（默认当前） | `{ok, entries:[{name, is_dir, size}]}` |
列目录内容 |
⚠
**异步铁律（同第 5 节）**：这些接口返回的是 Promise，KN 运行时不会 `await`。
积木 `function` 里要「fire-and-forget + toast」：启动异步任务后同步 `returnnull`，结
果稍后由 `window.__CUE_TOAST__` 显示。
```js
function: function (params) {
if
(window.pywebview
&&
window.pywebview.api.read_clipboard) {
window.pywebview.api
window.pywebview.api.read_clipboard().then(function (r) {
if (r && r.ok) window.__CUE_TOAST__('剪贴板: ' + r.text, 'ok');
else window.__CUE_TOAST__('读取失败: ' + (r && r.error), 'error');
});
} else {
window.__CUE_TOAST__('仅在桌面壳内可用', 'error');
}
return null;
}
```
// 同步返回，结果稍后由 toast 显示
**示例：读取 D 盘文件并显示前 200 字**
```js
window.pywebview.api.read_text_file('D:/data/config.txt').then(function (r) {
if (r && r.ok) window.__CUE_TOAST__('内容: ' + r.content.slice(0, 200), 'ok');
});
```
&&
> 路径写法：Windows 用正斜杠或双反斜杠，例如 `'D:/data/a.txt'` 或
`'D:\\\\data\\\\a.txt'`。
